# Detects whether a PDF looks like an existing session @staticmethod def calculate_text_similarity( text1: str, text2: str ) -> float: def tokenize(text): text = text.lower() text = ''.join( c if c.isalnum() or c.isspace() else ' ' for c in text ) words = set(text.split()) stopwords = {'the', 'a', 'is', 'of', ...} return words - stopwords set1 = tokenize(text1) set2 = tokenize(text2) if not set1 or not set2: return 0.0 intersection = set1 & set2 union = set1 | set2 return len(intersection) / len(union)
Called for the creation of a session to avoid re-generating existing questions.
Jaccard coeficient
J(A,B) = |A ∩ B| / |A ∪ B|
AText words 1
BText words 2
∩Words in common
∪All unique words
Why this algorithm?
Detects if a new PDF covers a topic already in the database
If score > threshold → reuses existing questions
Reduces unnecessary LLM calls → cost & latency

Biggest Challenge

Integrate AI into the application (Sprint 2)

I had underestimated the complexity of LLM calls, PDF extraction, and prompt engineering. This forced me to remove a key feature in order to deliver the MVP on time.

Key takeaway: Prioritize ruthlessly—a focused MVP delivers results, while an overly ambitious project does not.
